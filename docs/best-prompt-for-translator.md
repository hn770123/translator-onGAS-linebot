# AIモデルに翻訳させるときのベストプラクティス

## 1. 明確な指示を与える

### 基本的な構造
```
あなたは[言語A]から[言語B]への翻訳者です。
以下のテキストを翻訳してください:

[翻訳対象テキスト]
```

### より効果的な指示
- 翻訳の目的を明示する(ビジネス文書、技術文書、カジュアルな会話など)
- 対象読者を指定する(一般読者、専門家、子供向けなど)
- 翻訳のトーンを指定する(フォーマル、カジュアル、中立など)

## 2. 文脈を提供する

### 文脈の重要性
AIモデルは文脈を理解することで、より適切な翻訳を生成できます。

```
# 良い例
この文章は技術ドキュメントの一部です。
開発者向けのAPI説明文として翻訳してください:

[翻訳対象テキスト]

# 悪い例
翻訳してください:
[翻訳対象テキスト]
```

### 会話履歴の活用
- 連続した翻訳では、前の会話履歴を保持することで一貫性が向上
- 専門用語の訳し方を統一できる

## 3. 出力フォーマットを指定する

### 明確なフォーマット指示
```
以下のJSON形式で出力してください:
{
  "original": "元のテキスト",
  "translated": "翻訳されたテキスト",
  "language": "翻訳先の言語"
}
```

### マークダウンや特殊記号の保持
```
元のテキストのマークダウン記法やコードブロックを保持したまま翻訳してください。
```

## 4. 専門用語の扱い

### 用語集の提供
```
以下の用語は下記の通り翻訳してください:
- Authentication → 認証
- Authorization → 認可
- Token → トークン(カタカナのまま)

[翻訳対象テキスト]
```

### 固有名詞の扱い
```
人名、地名、製品名などの固有名詞は翻訳せず、元の表記のまま保持してください。
```

## 5. Few-shot Examples(例示)の活用

### 例を示すことで精度向上
```
以下の例のように翻訳してください:

例1:
原文: "The user needs to authenticate before accessing the resource."
翻訳: "ユーザーはリソースにアクセスする前に認証する必要があります。"

例2:
原文: "Invalid credentials provided."
翻訳: "無効な認証情報が提供されました。"

それでは、以下のテキストを翻訳してください:
[翻訳対象テキスト]
```

## 6. 翻訳の品質を高める指示

### 自然な表現の要求
```
直訳ではなく、[言語B]の自然な表現で翻訳してください。
意訳が必要な場合は、元の意味を保ちながら自然な表現を優先してください。
```

### 文化的配慮
```
文化的な背景や慣用句に注意して翻訳してください。
[言語B]の文化に適した表現に調整してください。
```

## 7. 長文の翻訳

### 分割翻訳
- 長文は適切な単位(段落、章など)で分割
- 各部分の翻訳後、全体の一貫性を確認

### 段階的アプローチ
```
ステップ1: まず全体を大まかに翻訳
ステップ2: 専門用語や重要な箇所を精査
ステップ3: 全体の流れと一貫性を確認して最終調整
```

## 8. エラーハンドリング

### 不明な部分の対処
```
翻訳が不明確な箇所や複数の解釈が可能な箇所は、
以下の形式で複数の候補を提示してください:

翻訳案1: [...]
翻訳案2: [...]
推奨: 翻訳案1(理由: ...)
```

## 9. システムプロンプトの活用

### ロールの明確化
```
あなたは10年以上の経験を持つプロの翻訳者です。
[専門分野]に精通しており、正確で自然な翻訳を提供します。
```

### 制約の明示
```
以下の制約に従ってください:
- 翻訳のみを出力し、説明や注釈は含めない
- 元のテキストの改行や段落構造を保持する
- HTMLタグやマークダウン記法はそのまま保持する
```

## 10. 翻訳の検証と改善

### セルフチェックの促進
```
翻訳後、以下の点を確認してください:
1. 元の意味が正確に伝わっているか
2. 自然な表現になっているか
3. 専門用語が正しく使われているか
4. 文法的に正しいか

確認後、最終的な翻訳を出力してください。
```

### 逆翻訳による確認
```
翻訳後、その翻訳を元の言語に逆翻訳して、
意味が保持されているか確認してください。
```

## まとめ

効果的な翻訳プロンプトの要素:
1. **明確性**: 翻訳の目的、対象読者、トーンを明示
2. **文脈**: 十分な背景情報を提供
3. **構造化**: 出力フォーマットを明確に指定
4. **例示**: Few-shot examplesで期待する品質を示す
5. **制約**: 守るべきルールを明確に記述
6. **検証**: 品質確認のステップを含める

これらのベストプラクティスを組み合わせることで、
AIモデルによる高品質な翻訳を実現できます。
